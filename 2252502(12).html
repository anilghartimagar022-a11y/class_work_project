Chapter-4

LANからインターネットへの接続

🌐 LAN からインターネットへ 接続する仕組み

LAN 内の PC やスマホがインターネットに接続できるのは、
ルーターが LAN と WAN（インターネット）をつなぐ役割をしているからです

🏠 1. LAN（ローカルネットワーク）とは

LAN は、家・学校・オフィスなどの限られた範囲のネットワークです。
LAN 内の機器は、スイッチや Wi‑Fi アクセスポイントを通じて互いに通信します。
検索結果でも、LAN は「建物やキャンパスなどの小さな範囲でデバイスを接続するネットワーク」と説明されています

🔌 2. LAN とインターネットをつなぐのは「ルーター」

LAN の外側には WAN（インターネット） が広がっています。
LAN と WAN の境界にあるのが ルーターです。
ルーターは次の役割を持ちます：
✔ IP アドレス変換（NAT）
LAN 内のプライベート IP（例：192.168.x.x）を、
インターネット用のグローバル IP に変換します。
✔ ルーティング
LAN → インターネット
インターネット → LAN
の通信経路を決めます。
✔ DHCP
LAN 内の PC に IP アドレスを自動配布します。

🌍 3. インターネット接続の流れ（超シンプル版）

① PC が LAN に接続
有線（LAN ケーブル）または Wi‑Fi
。 ② ルーターから IP アドレスをもらう（DHCP）
③ PC が外部サイトへアクセス要求
例：YouTube を開く
④ ルーターが NAT を使って外へ送る
LAN 内のプライベート IP → グローバル IP に変換。
⑤ インターネットからの応答をルーターが受け取る
ルーターは NAT テーブルを見て、
どの PC へ返すべきか判断。
⑥ PC にデータが届く
YouTube が表示される。

🧩 4. LAN → インターネット接続に必要な機器

機器 役割
PC / スマホ LAN に接続してインターネットを利用する端末
スイッチ LAN 内の複数の機器を有線で接続する
Wi‑Fi アクセスポイント LAN 内の機器を無線で接続する
ルーター LAN とインターネットをつなぐ。NAT・DHCP・ルーティングを担当
モデム ISP（インターネットサービスプロバイダ）と接続するための機器

🎓 まとめ（覚えるべきポイント）

  • LAN は家や学校などの小さなネットワーク
  • インターネットへ出るには ルーターが必須
  • ルーターは NAT・ルーティング・DHCP を担当
  • LAN 内の PC はプライベート IP、外はグローバル IP
  • モデム → ルーター → スイッチ/Wi‑Fi → PC の順で接続

🔥 ファイアウォールとは？

ファイアウォール（Firewall）とは、ネットワークを守るための“防御システム” のことだよ。

⇢簡単に言えば、安全なネットワーク（例：自分のPCや大学のネットワーク）と、危険が潜む外部ネットワーク（例：インターネット）との間に置く“門番”。 外から来る通信、内側から外へ出ていく通信を監視して、 許可するか、ブロックするかを判断する仕組み なんだ。

🛡️ ファイアウォールの役割

  • 不正アクセスを防ぐ
  • マルウェアや攻撃の侵入を防ぐ
  • 許可された通信だけを通す
  • ネットワークの安全性を高める

🧩 種類

種類 説明
ネットワーク型ファイアウォール ルーターや企業ネットワークに設置されるタイプ
ホスト型ファイアウォール WindowsやMacに入っているPC単体を守るタイプ
Webアプリケーションファイアウォール（WAF） Webサイトを攻撃から守る専門タイプ

📦 パケットフィルタリングとは

（Packet Filtering）

パケットフィルタリングは、通信データ（パケット）の“ヘッダ情報”を見て通すかブロックするかを判断する仕組み だよ。 検索結果でも、

• パケットのヘッダ（IPアドレス、ポート番号、プロトコル）をチェックして許可/拒否を決める と説明されている。

🔍 何をチェックする？

• 送信元IPアドレス

• 宛先IPアドレス

• ポート番号（80, 443 など）

• プロトコル（TCP, UDP など）

🛡️ 特徴

  • 高速（ヘッダだけ見るため処理が軽い）
  • 基本的な防御に向いている
  • ただし、パケットの中身（コンテンツ）は見ないため、高度な攻撃は防ぎにくい
---

🌐 URLフィルタリングとは

（URL Filtering）

URLフィルタリングは、アクセスしようとしている Web サイトの URL を基準に、アクセスを許可・拒否する仕組み。 検索結果では、

• URL に基づいてアクセスを制御し、悪意のあるサイトや不適切なサイトをブロックする技術 と説明されている。

🔍 何をチェックする？

• URL（例：https://example.com/malware）

• ドメイン名（例：example.com）

• カテゴリ（例：アダルト、ギャンブル、マルウェアサイトなど）

• レピュテーション（安全性スコア）

🛡️ 特徴

  • 危険なサイトへのアクセスを防ぐ
  • 企業・学校のポリシー管理に便利
  • リアルタイムで悪意あるサイトを判定できる
🎯 超シンプルまとめ
• パケットフィルタリング
→ 「IP・ポートで通信を通すか決める門番」
• URLフィルタリング
→ 「危険なWebサイトへのアクセスを止めるフィルター」
